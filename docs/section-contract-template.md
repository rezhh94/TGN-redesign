# Section Contract Template

Before coding a new section, fill this out.

## Section name

Example:
01 / Approach Statement Bridge

## Role in page journey

What does this section do after the previous section?

## One idea

What is the single idea this section should communicate?

## Focal point

What should the user notice first?

## Tone

Dark / light / mixed?
Quiet / dense / cinematic / structured?

## Layout archetype

Examples:
- full-screen hero
- centered statement
- editorial split
- vertical interactive list
- horizontal proof gallery
- sticky process
- compact footer index

## Why this layout is different from the previous section

Explain how this avoids repetition.

## Content

Exact copy, headings, labels and links.

## Must include

Required text, links or SEO elements.

## Must not include

Specific exclusions.

## Motion plan

No motion in the first build unless explicitly approved.

Future motion idea:
- reveal
- text fill
- line draw
- hover
- scroll highlight

## Static acceptance criteria

What must be true before motion is added?

## Files expected to change

List expected files only.

## Validation

Build/typecheck/diff-check required.
